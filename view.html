Topic 1-2

Views & ViewGroups

Definition of a View

  • A rectangular area on the screen.
  • Has a width, height, and sometimes a background color.
  • Key Concept: The View on the screen is drawn by a Java Object inside the Android device. The Java object is the "real" View.

The ViewGroup

A ViewGroup is a big View—often invisible—that acts as a container.

  • It positions smaller Views inside it.
  • Creates a Parent-Child relationship.

Topic 5-6

XML Basics & Syntax

XML (Extensible Markup Language) describes the layout in a "Family Tree" structure. An Android layout always consists of one root View which may contain smaller ones.

XML Tag Rules

1. Opening & Closing Tags

Used for elements that contain others (like LinearLayout).

<LinearLayout> ... </LinearLayout>
2. Self-Closing Tags

Used for elements that don't hold content (like TextView).

<TextView ... />
LinearLayout Example (Vertical)
<!-- Parent: LinearLayout --> <LinearLayout android:layout_width="match_parent" android:layout_height="match_parent" android:orientation="vertical" android:background="#E0E0E0"> <!-- Child 1 --> <TextView android:text="Title" android:layout_width="wrap_content" android:layout_height="wrap_content" /> <!-- Child 2 --> <TextView android:text="Subtitle" android:layout_width="wrap_content" android:layout_height="wrap_content" /> </LinearLayout>

Units: dp vs sp

Respecting user preferences in Settings.

sp (Scale-Independent Pixels)

Definition: A unit of length for specifying the size of a font.
Behavior: Its length depends on the user's preference for font size set in the Settings app. Always use `sp` for text.

dp (Density-Independent Pixels)

Definition: A unit for layout dimensions (width, height, padding).
Behavior: Consistent physical size across screens. Does NOT scale with font settings.

ImageView & Performance

Use ImageView to display images (icons/photos). You must place image files in the res/drawable folder.

AttributeDescription
